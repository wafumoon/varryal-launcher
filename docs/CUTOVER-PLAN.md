# Cutover-план (упрощённый): новый Tauri-лаунчер для админов

> **Контекст:** лаунчер в DEV, им пользуются **только админы**, реальных игроков на нём нет.
> Значит ломать некого — делаем **простую атомарную замену**, без dual-track, окон и осторожностей.

## Суть
Старый JavaFX-клиент заменяем новым Tauri-лаунчером «в лоб». Откат тривиален (бэкап есть).

## Шаги

### 1. Сервер: `Varryal.jar` → bridge (постоянно)
- В `modules.json` на LaunchServer: `loadLauncherModules` — заменить `JavaRuntime.jar` → `VarryalRuntime.jar` (постоянно, не временно).
- `binaryName` оставить `Varryal`.
- Залить актуальный `VarryalRuntime.jar` (наш bridge), **Restart** сервера, в консоли **`build`**.
- Результат: `updates/Varryal.jar` (и `Varryal.exe`) становятся bridge-сборкой (подписанной, proguard). JavaFX-рантайм выведен.
- Бэкап текущего — в `_bridgetest-backup/` (есть). Откат = вернуть `Varryal.jar`/`modules.json` + restart.

### 2. Раздача новой оболочки
- Оболочка уже тянет `https://launcher.varryal.ru/Varryal.jar` (в `runner.rs`) — после шага 1 это и есть bridge. **Код оболочки менять не нужно.**
- Взять CI-инсталляторы (`.exe`/`.msi`/`.dmg`/`.AppImage`/`.deb`) и раздать админам (ссылка/файл — как удобно; сайт пока не обязателен).

### 3. Живой end-to-end тест (админ)
- Админ ставит новую оболочку → она качает JRE 25 → тянет `Varryal.jar` (bridge) → **браузер-логин → выбор персонажа → выбор сервера → скачивание клиента (хэш) → запуск игры** на `play.varryal.ru`.
- Это первый живой прогон связки download→launch (раньше проверяли только auth/profiles).

## Решения (можно отложить, ядро cutover от них не зависит)
1. **macOS:** сейчас сборка только `aarch64` (Apple Silicon). Добавлять universal (Intel) — да/нет/потом.
2. **Авто-апдейтер оболочки:** включить (minisign-ключи + `latest.json`) — сейчас/потом.
3. **Где хостить инсталляторы:** рядом с LaunchServer / GitHub Releases / просто файлом админам.
